# 데이터 Validation 프로그램 PRD

## 1. 제품 개요

### 1.1 제품명

**Maintenance Data Validation & Error Report System**

### 1.2 목적

해양플랜트 장비 및 부품의 유지보수 관련 정보를 엑셀로 수집할 때, 입력 데이터의 정합성 오류를 자동으로 검증하고, 오류 내용을 담당자별 리포트로 생성하여 메일로 전달하는 프로그램을 구축한다.

### 1.3 배경

현재 유지보수 관련 데이터를 여러 담당자로부터 엑셀 파일 형태로 수집하고 있으나, 데이터 형식 불일치, 필수값 누락, 범위 오류, 표준 목록 외 값 입력 등으로 인해 데이터 정합성 문제가 자주 발생하고 있다.

이로 인해 검토자가 수작업으로 오류를 찾아 담당자에게 코멘트를 전달해야 하며, 반복 업무와 커뮤니케이션 비용이 많이 발생한다.

본 프로그램은 엑셀 업로드, 검증 규칙 설정, 자동 검증, 담당자별 오류 리포트 생성, 메일 발송 기능을 제공하여 데이터 품질 관리 업무를 효율화하는 것을 목표로 한다.

## 2. 주요 사용자

### 2.1 관리자 / 검토자

- 엑셀 데이터를 업로드하는 사용자
- 컬럼별 검증 규칙을 설정하는 사용자
- 검증 결과를 확인하고 담당자에게 리포트를 발송하는 사용자
- 전체 데이터 품질 현황을 관리하는 사용자

### 2.2 데이터 입력 담당자

- 장비 또는 부품 유지보수 정보를 작성하는 사용자
- 오류 리포트를 메일로 수신하는 사용자
- 오류 내용을 확인하고 데이터를 수정하여 재제출하는 사용자

## 3. 핵심 문제

### 3.1 현재 문제점

- 수집된 엑셀 데이터의 정합성이 맞지 않음
- 컬럼별 데이터 타입이 일관되지 않음
- 필수 입력값이 누락되는 경우가 있음
- 숫자 데이터가 허용 범위를 벗어나는 경우가 있음
- 표준 목록에 없는 값이 입력되는 경우가 있음
- 오류 검토 및 코멘트 작성이 수작업으로 이루어짐
- 담당자별 오류 내용을 분리하여 전달하는 데 시간이 많이 소요됨

### 3.2 해결 방향

- 엑셀 업로드 후 자동으로 데이터 검증
- 컬럼별 검증 기준을 사용자가 직접 설정
- 오류 발생 위치와 원인을 자동으로 표시
- 담당자별 오류 리포트 자동 생성
- 담당자 메일 주소 기준으로 오류 리포트 자동 발송

---

## 4. 제품 목표

### 4.1 업무 목표

- 데이터 검증 업무의 수작업 감소
- 데이터 오류 검출 정확도 향상
- 담당자별 오류 피드백 자동화
- 유지보수 데이터 품질 향상
- 반복적인 검토 및 메일 작성 시간 단축

### 4.2 정량 목표

- 수작업 검증 시간 50% 이상 절감
- 담당자별 오류 리포트 작성 시간 70% 이상 절감
- 필수값 누락, 타입 오류, 범위 오류 등 기본 오류 자동 검출률 90% 이상 달성
- 동일한 검증 기준을 반복 적용할 수 있는 Rule Template 구축

## 5.3 검증 규칙 설정 기능

### 설명

사용자는 컬럼별로 데이터 검증 기준을 설정할 수 있다.

### 요구사항

컬럼별로 아래 검증 기준을 설정할 수 있어야 한다.

### 5.3.1 필수값 여부

- 해당 컬럼이 반드시 입력되어야 하는지 설정
- 빈 값, 공백만 있는 값, NULL 값 검출

예시:

| 컬럼명 | 규칙 |
|---|---|
| Equipment Tag No. | 필수 입력 |
| Responsible Email | 필수 입력 |

### 5.3.2 데이터 타입 검증

컬럼별 데이터 타입을 지정할 수 있어야 한다.

지원 데이터 타입:

- Text
- Number
- Integer
- Decimal
- Date
- Email
- Boolean
- Code/List
- URL

예시:

| 컬럼명 | 데이터 타입 |
|---|---|
| Maintenance Cycle | Number |
| Responsible Email | Email |
| Inspection Date | Date |

### 5.3.3 숫자 범위 검증

숫자형 컬럼에 대해 최소값, 최대값을 설정할 수 있어야 한다.

예시:

| 컬럼명 | 최소값 | 최대값 |
|---|---:|---:|
| Maintenance Cycle | 1 | 120 |
| Operating Hour | 0 | 999999 |

검증 예시:

- Maintenance Cycle 값이 0이면 오류
- Operating Hour 값이 음수이면 오류

### 5.3.4 정형 목록 검증

특정 컬럼에는 미리 정의된 목록 값만 입력되도록 설정할 수 있어야 한다.

예시:

| 컬럼명 | 허용값 |
|---|---|
| Maintenance Type | Preventive, Corrective, Predictive |
| Criticality | High, Medium, Low |
| Status | Open, In Progress, Closed |

검증 예시:

- Criticality 컬럼에 `Very High` 입력 시 오류
- Maintenance Type 컬럼에 `PM` 입력 시 허용 목록에 없으면 오류

### 5.3.5 날짜 형식 검증

날짜 컬럼에 대해 날짜 형식과 유효성을 검증할 수 있어야 한다.

검증 항목:

- 날짜 형식 여부
- 미래일 허용 여부
- 과거일 허용 여부
- 특정 기준일 이전/이후 여부

예시:

| 컬럼명 | 규칙 |
|---|---|
| Inspection Date | 날짜 형식 필수 |
| Next Maintenance Date | 오늘 이후 날짜만 허용 |

### 5.3.6 이메일 형식 검증

담당자 이메일 컬럼은 이메일 형식인지 검증한다.

검증 예시:

- `abc@company.com` → 정상
- `abc.company.com` → 오류
- `abc@` → 오류

### 5.3.7 중복값 검증

특정 컬럼 또는 컬럼 조합에 대해 중복 여부를 검증할 수 있어야 한다.

예시:

- Data ID 중복 불가
- Equipment Tag No. + Part No. 조합 중복 불가

### 5.3.8 조건부 검증

특정 컬럼 값에 따라 다른 컬럼의 필수 여부나 허용값을 다르게 설정할 수 있어야 한다.

예시:

- Maintenance Type이 `Preventive`이면 Maintenance Cycle은 필수
- Criticality가 `High`이면 Inspection Interval은 12개월 이하
- Status가 `Closed`이면 Close Date는 필수

## 5.6 오류 결과 화면 기능

### 설명

검증 완료 후 오류 데이터를 화면에서 확인할 수 있다.

### 요구사항

- 전체 오류 목록 표시
- 담당자별 필터
- 컬럼별 필터
- 오류 유형별 필터
- Data ID 검색
- 원본 데이터 행 보기
- 오류 메시지 자동 생성
- 엑셀 다운로드 기능 제공

### 오류 메시지 예시

| 오류 유형 | 메시지 예시 |
|---|---|
| 필수값 누락 | Equipment Tag No.는 필수 입력 항목입니다. |
| 데이터 타입 오류 | Maintenance Cycle은 숫자 형식이어야 합니다. |
| 범위 오류 | Maintenance Cycle은 1~120 사이의 값이어야 합니다. |
| 목록 오류 | Criticality는 High, Medium, Low 중 하나여야 합니다. |
| 이메일 오류 | Responsible Email 형식이 올바르지 않습니다. |
| 중복 오류 | 동일한 Data ID가 중복 입력되었습니다. |

## 5.7 담당자별 리포트 생성 기능

### 설명

검증 결과를 담당자별로 분리하여 오류 리포트를 생성한다.

### 요구사항

- Responsible Person 기준 리포트 생성
- Responsible Email 기준 리포트 생성
- 담당자별 오류 항목만 포함
- 리포트 파일 형식은 Excel 또는 PDF로 생성
- 리포트 파일명 자동 생성

### 리포트 파일명 예시

```text
Validation_Report_홍길동_2026-06-27.xlsx
```

### 리포트 구성

1. 리포트 요약
2. 담당자 정보
3. 오류 건수 요약
4. 오류 상세 목록
5. 수정 요청 코멘트
6. 재제출 안내 문구

### 리포트 예시 구성

| Data ID | Row No. | Column | Input Value | Error Type | Comment |
|---|---:|---|---|---|---|
| EQ-001 | 15 | Maintenance Cycle | ABC | Type Error | 숫자 형식으로 입력해 주세요. |
| EQ-002 | 21 | Criticality | Very High | List Error | High, Medium, Low 중 하나로 입력해 주세요. |


```text

- 허용 목록 외 값 입력

감사합니다.
```

## 7. 사용자 흐름

10. 담당자별 이메일 발송 내용을 확인한다.

## 14. 개발 시 고려사항

### 14.1 엑셀 포맷 변경 대응

업로드되는 엑셀 파일의 컬럼명이 변경될 수 있으므로, 컬럼명을 고정하지 않고 매핑 방식으로 관리해야 한다.

### 14.2 담당자 정보 기준

담당자별 리포트 생성을 위해 최소한 아래 컬럼은 반드시 지정되어야 한다.

- Data ID
- Responsible Person
- Responsible Email

해당 정보가 누락된 경우 별도 오류 항목으로 분류해야 한다.

### 14.3 오류 메시지 관리

오류 메시지는 시스템 기본 메시지를 제공하되, 사용자가 컬럼별로 수정할 수 있도록 한다.

예시:

- 기본 메시지: “값이 허용 범위를 벗어났습니다.”
- 사용자 정의 메시지: “Maintenance Cycle은 1개월 이상 120개월 이하로 입력해 주세요.”

### 14.4 원본 데이터 보존

검증 과정에서 원본 업로드 파일은 변경하지 않아야 한다. 오류 리포트는 별도 파일로 생성한다.

### 14.5 재검증 프로세스

담당자가 수정된 파일을 재제출하면, 동일한 Rule Template으로 다시 검증할 수 있어야 한다.
